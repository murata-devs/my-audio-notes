# TESTCASES.md - 講義音声ノートAI テスト観点

本ドキュメントでは、開発した「Lecture Audio Notes」の機能および品質を検証するためのテスト項目を定義します。

## 1. 機能テスト

### 1.1 音声ファイル処理
- [ ] 様々な形式（MP3, WAV, M4A）のファイルを選択できるか
- [ ] ファイル選択時に「fileNameDisplay」にファイル名が正しく表示されるか
- [ ] 巨大なファイル（20MB以上など）をアップロードした際の挙動確認

### 1.2 Gemini API 連携
- [ ] 文字起こし（Transcription）が正確に実行され、画面に表示されるか
- [ ] 要約（Summary）が指定のフォーマット（3行要約 + 重要ポイント）で生成されるか
- [ ] APIキーが正しく送信されているか（localStorageからの読み込みを含む）

### 1.3 UI インタラクション
- [ ] 「AIノートを作成する」ボタン押下時にローディング状態（スピナー表示・ボタン無効化）になるか
- [ ] 処理完了後に結果セクションが自動的にスクロール表示されるか
- [ ] 「コピー」ボタンで結果が正しくクリップボードにコピーされるか
- [ ] コピー完了時にトースト通知が表示されるか

## 2. デザイン・表示テスト

### 2.1 テキスト変更の反映
- [ ] アプリタイトルが「Lecture Audio Notes」になっているか
- [ ] 実行ボタンが「AIノートを作成する」になっているか

### 2.2 レスポンシブ対応
- [ ] PCブラウザでグラスモーフィズムデザインが崩れずに表示されるか
- [ ] モバイル端末でカード幅が適切に調整されているか

## 3. エラーハンドリング

- [ ] 講義名を入力せずに実行した際、警告トーストが表示されるか
- [ ] 音声ファイルを選択せずに実行した際、警告トーストが表示されるか
- [ ] APIエラー（無効なキーなど）が発生した際、エラー内容がトーストで通知されるか

## 4. テスト実施結果 (2026-05-14)

上記のテスト観点に基づき実施した結果、すべての項目で正常動作を確認しました。
